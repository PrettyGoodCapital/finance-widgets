Providers

This library is in pre-alpha stage and is not yet suitable for production usage.

Providers bind data sources/sinks to widgets. They leverage Lit's Context in order to manage a subtree of components.

Providers can implement one or more sets of interfaces, which themselves can be applied to any number of widgets.

  • Single Component Provider (see Components) - Provides data about a single asset
  • Portfolio Component Provider (see Components) - Provides a collection of data, such as quotes for all assets in a basket
  • Single Component Target - Receives a single piece of data, such as a single order to place
  • Portfolio Component Target - Receives a collection of data, such as a basket of orders to place

Several providers are given for demo purposes, such as the RandomProvider. In general, end users should write their own providers to bind their APIs and data sources to the widgets.

Some financial data vendors provide robust Data APIs, so this library supports a few of those, such as the PolygonProvider.
